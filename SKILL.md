## 核心理念：慢整理 (Slow Organizing) 与 场景化规划

**“整理看得见的物品，改变看不见的世界。”**
1. **慢整理**：不追求一次性永不复乱，通过“0+4 模式”建立可持续秩序。
2. **MECE 原则**：分类的科学标准 —— “相互独立，完全穷尽”。
3. **动线与频率原则 (Accessibility)**：
   - **黄金位置 (Golden Zone)**：视线与腰部之间，触手可及。存放【高频使用】物品。
   - **收纳逻辑**决定位置。
4. **场景化规划 (Activity-based Planning)**：
   - **核心逻辑**：空间除了区分频率，更要定义其**“主体活动”**。

## 数据操作避坑与分类经验

- **JSON 数据结构陷阱**：遍历 `data.get("items", [])` 而非直接遍历 `data`。
- **物品分类边界案例**：
  - **空包装/空盒子**：属于 `🏠 家居与工具 -> 包装与收纳`。
  - **蒸汽眼罩**：属于 `🧼 个人护理 -> 身体洗护`。

## 11. 飞书多维表格 (Feishu Base) 深度集成

### 11.1 唯一标识与同步强一致性
1. **本地 ID (`id`)**: 唯一准则。
2. **唯一入口**: `/scripts/sync_final.py`。

## 12. 同步性能与内容一致性 (Sync Performance & Content Integrity)

### 12.1 深度比对逻辑 (Deep Compare)
1. **内容指纹校验**：在向飞书推送更新前，系统必须进行“深度内容比对”。
2. **核心字段范围**：`物品名称`、`位置`、`容器`、`大类`、`子分类`、`状态`、`备注`。
3. **静默跳过策略**：若内容一致，即使本地时间较新，也应**跳过** API 请求，保护飞书“更新时间”真实性。

## 13. 历史数据迁移与实物对照逻辑 (Historical Data Migration)

### 13.1 事实驱动原则
1. **禁脑补列表**：严禁通过分类联想自行添加源文件中不存在的物品。
2. **子类导向迁移 (Sub-category Led)**：以新系统子类为锚点，去历史库中打捞匹配项。
3. **实物找寻与安置 (Physical Interaction)**：迁移是找回实物并赋予新家（位置/容器）的过程。
4. **位置与容器联动 (Container-Location Linkage)**：移动某个容器时，AI 自动批量更新其内部所有物品的位置。
